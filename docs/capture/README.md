# Re-capturing user-guide screenshots

The screenshots in [`docs/user-guide.md`](../user-guide.md) are generated by
a two-step pipeline against a running Roundhouse stack.

```bash
# 1) Seed Atlas Shrugged demo servers into the running stack.
#    Idempotent: prior demo servers are deleted first.
python3 docs/capture/seed_demo.py

# 2) Capture every notable view to PNG, in both themes.
node docs/capture/capture.mjs --theme both

# (Optional) Drop the demo servers when finished.
python3 docs/capture/seed_demo.py --cleanup
```

Output lands in `docs/screenshots/{dark,light}/`.

## Prerequisites

- A running Roundhouse stack reachable at `http://localhost:3080` (override
  with `--base`). The default admin (`admin@mcp.local` / `admin`) must
  exist; override with `--email` / `--password`.
- Node 18+ and Playwright (`npm install` inside `docs/capture/` if
  `node_modules` is missing, then `npx playwright install chromium`).
- Python 3 with `curl` on `PATH` (the seed script shells out to curl
  because Python's `urllib` trips this stack's auth middleware in a way we
  never tracked down — `curl` works identically).

## Demo server cast

The screenshots feature seven Atlas Shrugged-themed servers chosen to
exercise every editor surface:

| Server | Mode | Role in the docs |
|--------|------|------------------|
| `taggart-transcontinental` | structured | Flagship server — covers every editor tab (tools, resources, prompts, env, auth, assets, usage). |
| `rearden-metal`            | structured | Pip-package and resource-template examples. |
| `danconia-copper`          | structured | Stopped state — shows the gray badge. |
| `wyatt-oil`                | code       | Code-mode source editor and code-mode env tab. |
| `galt-engine`              | structured | `LOG_LEVEL=DEBUG` set, so the Logs tab dropdown is exercised. |
| `mulligan-bank`            | structured | Stopped, more env-var variety. |
| `stockton-foundry`         | structured | Round-out the dashboard fleet view. |

## Re-running after a UI change

`capture.mjs` is intentionally selector-loose (text patterns + roles rather
than fragile CSS selectors). If a step fails after a rename, edit the `wait`
entry for that step in the `STEPS` array.

To capture only one theme during iteration:

```bash
node docs/capture/capture.mjs --theme dark
```

To skip the seed (already done in your session):

```bash
node docs/capture/capture.mjs --theme dark
```
